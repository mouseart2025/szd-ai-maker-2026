## 四、什么是 Vibe Coding

**Vibe Coding（氛围编程）** 由 Andrej Karpathy（OpenAI 联合创始人、前 Tesla AI 总监）2025 年提出：

> "你完全可以沉浸在'感觉'里，看效果、说修改、再看效果，忘掉代码本身的存在。"

| 传统编程 | 聊天式 AI | **Vibe Coding** |
|---|---|---|
| 学语法、记函数 | 网页问 AI | 写需求 / 描述 |
| 自己写代码 | 复制 AI 代码 | **AI 自动写代码** |
| 自己编译上传 | 粘贴到 IDE，手动上传 | **AI 自动编译上传** |
| 报错自己查 | 复制粘贴报错来回 | **AI 自己读报错自己修** |
| 你是程序员 | 你是传话筒 | **你是导演** |

**一句话总结：未来的关键能力不是写代码，是描述需求**。

## 五、你的硬件套件

每人一套，培训结束后归还。

| 组件 | 说明 |
|---|---|
| XIAO ESP32S3 Sense | 主控（拇指大小，内置摄像头 + 麦克风 + WiFi）· **主板上有一颗小黄色 LED 是今天的演示主角** |
| Grove Shield for XIAO | 扩展板，免焊接 |
| Grove Mini 风扇模块 | 接 **D1** 端口 |
| Grove 蜂鸣器模块 | 接 **D2** 端口 |
| USB-C 数据线 | 供电 + 上传程序 |

### 4 步接线

```
Step 1 · XIAO 插入 Grove Shield（针脚对齐，按紧）
Step 2 · Grove Mini 风扇 → D1 端口
Step 3 · Grove Buzzer → D2 端口
Step 4 · USB-C 数据线接电脑
```

**口诀：D1 = 风扇 · D2 = Buzzer**（LED 演示用 XIAO 主板上的板载 LED, 不需要外接）

XIAO 上小黄灯亮起 = 供电正常。

### 接不上 / 灯不亮的 3 种典型原因

- ❌ USB-C 线**只能充电不能传数据** → 换一根（讲师桌上有备用）
- ❌ XIAO 没插紧 Grove Shield → 用力按下去
- ❌ 端口接错 → 对照 D1 / D2 检查一遍

搞不定直接举手找助教。

```bash
opencode --version
```

显示版本号（如 `0.x.x`）= 装好了。

## 八、实践任务 · 3 步走（你动手）

**规则**：3 个任务都在**同一个项目**上叠加，不要重新开始。让你体验"持续迭代"的工作流。

### 任务 1 · 点亮板载 LED（5 min）

> 💡 **关于今天的"LED"**: 我们用 XIAO 板上自带的**小黄色 LED**（板载 LED, GPIO21）做演示。它就在你桌上 XIAO 主板上, 不需要接任何外设, USB 一插就能控制。

把下面这段话发给 AI：

```
请在 szd-ai-maker-2026 目录下创建一个新项目叫 my-experiments，在里面写一个程序：

让 XIAO 板载的 LED (LED_BUILTIN, GPIO21) 持续亮着（不要闪烁, 就是一直亮着）。

注意: XIAO 板载 LED 是反逻辑——digitalWrite(LED_BUILTIN, LOW) 才点亮。

编译并上传到我桌上的 XIAO。
```

**成功标志**：XIAO 主板上那颗**小黄色 LED** 一直亮着。

### 任务 2 · 加 Buzzer 提示音（8 min）

在**同一个对话**里继续说：

```
在同一个项目上加功能：

让 D2 端口的 Grove 蜂鸣器每 2 秒响一次短促的提示音（只响 100ms 然后停）。

板载 LED 保持持续亮。
```

**成功标志**：板载 LED 持续亮 + Buzzer 每 2 秒响一下。

### 任务 3 · 加风扇随机开关（12 min）

```
在同一个项目上再加：

让 D1 端口的风扇随机开关——每次随机转 3-10 秒，停 5-15 秒，循环。

板载 LED 和 Buzzer 保持原状。
```

**成功标志**：3 个硬件同时工作（板载 LED 持续亮 + Buzzer 每 2 秒响 + 风扇随机开关）。

> ⚠️ **关于风扇**：Grove Mini 风扇也是 5V 设计，在 3.3V 下扭矩偏弱，**可能转得慢或断断续续**，这是供电问题不是代码问题。能转动 = 任务完成。

### 提前完成的老师

自己想一个功能让 AI 加上。例如：

```
让板载 LED 在风扇转的时候变成快闪
```

或者换成你自己的想法——这才是 Vibe Coding 的自由。

## 九、BMAD 实战 · 用手机控制硬件（全场高潮）

### BMAD 是什么

**让一个 AI 分步扮演不同角色帮你做项目**：

```
你的想法
    ↓
📋 PM 产品经理    → 想清楚"做什么"
    ↓
🎨 UX 设计师      → 想清楚"长什么样"
    ↓
🏗️ 架构师         → 想清楚"怎么搭"
    ↓
💻 开发者         → 写代码、编译、上传
    ↓
🧪 测试 + 迭代    → 跑通了吗？哪里要改？
```

和你们做产品设计的流程一样——先想用户、再画界面、再想结构、再动手做、再测试。AI 也应该按这个顺序来。

### 接下来 35 分钟 · 跟讲师同步走 4 步

**所有步骤都在同一个 OpenCode 对话里完成**，不要中途开新对话——AI 会丢掉上下文。

#### Step 1 · 📋 PM 角色（5 min）

```
现在请你扮演产品经理 (PM)。

我想做一个"老师在课堂上用手机控制桌上 LED 和风扇"的应用。

帮我写一份 PRD：
1. 用户场景（老师什么情况下会用？）
2. 核心功能（P0: 控制 LED 开关 + 控制风扇开关 + 看到当前状态）
3. 用户操作流程
4. 这个 MVP 在 20 分钟内做完，请控制复杂度。
```

#### Step 2 · 🎨 UX 设计师（5 min）

```
现在请你切换为 UX 设计师角色。

基于刚才的 PRD，设计这个手机网页的界面：
1. 页面布局（手机竖屏）
2. LED 按钮和风扇按钮的位置、大小、颜色
3. 状态显示（开/关怎么显示？）
4. 视觉风格（配色？大字？圆角？背景？）
5. 响应式布局 · 适配手机。
```

#### Step 3 · 🏗️ 架构师（3 min）

```
现在请你切换为系统架构师角色。

基于以上 PRD 和 UX，告诉我技术上怎么实现：
1. XIAO 怎么开 WiFi AP？
2. 怎么开 Web Server？
3. 网页和 XIAO 之间怎么通信？（路由设计）
4. 用什么库？需要装吗？
5. 关键代码结构（setup / loop / handler）。
```

#### Step 4 · 💻 开发者（8 min）

```
现在请你切换为开发者角色。

基于以上 PRD、UX 和架构，写完整代码并上传到我的 XIAO。

WiFi 名建议用 "XIAO-Teacher-[你的姓氏]" 避免和邻桌冲突，密码 12345678。
```

AI 会自动写代码 → 编译 → 上传 → 串口输出 WiFi 名和 IP 地址。

### 🎉 手机连接体验（5 min）

1. 手机 WiFi 设置 → 找到 **XIAO-Teacher-[你姓氏]**
2. 输入密码 **12345678** → 连上
3. 浏览器打开 **http://192.168.4.1**
4. 看到你刚才设计的网页 → 点 LED 按钮 → **桌上的 LED 亮了**
5. 点风扇按钮 → **风扇转了**

> **整个应用**（网页 + 服务器 + WiFi 配置 + 硬件控制）**从你和 AI 的 4 段对话中自动生成**。全程你没写一行代码。

**想和邻桌玩**：连他的 WiFi，控制他桌上的硬件——这就是物联网。

## 十、扩展体验 · 零代码 AI 视觉

换一种 AI 赋能方式——不写需求，不写代码，纯图形界面操作。

**浏览器打开**：https://sensecraft.seeed.cc

### Step 1 · 部署预制模型（4 min）

1. USB 重新连接 XIAO
2. 选"预制模型" → "手势识别"
3. 一键部署到 XIAO
4. 对着摄像头比手势 → 看识别结果

### Step 2 · 训练你自己的 AI 模型（12 min）

1. **采集**：对摄像头拍"举手"图片 20 张 + "放下"图片 20 张
   - 也可以自定义：竖大拇指 / 比心 / OK 手势——什么都行
2. **训练**：点"开始训练" → 等约 3-5 分钟
3. **部署**：将训练好的模型部署到 XIAO → 实时测试识别效果

### Step 3 · 模型驱动硬件（4 min）

让 AI 帮你写代码，把视觉模型接到硬件：
- 识别"举手" → LED 亮
- 识别"放下" → 风扇启动

> **Vibe Coding = 写需求 · AI 写代码**
> **零代码视觉 = 采数据 · AI 训模型**
>
> 两种 AI 赋能路径，你的课上可以选适合的那种。

**卡住了**？SenseCraft 界面第一次用，迷路 2 分钟就举手——助教陪你走一遍，不要自己在平台里闷头找。

## 十二、反思实践 · 如何引入你的教学体系

**今天最值钱的 20 分钟**。不是学新东西——是想清楚**回去怎么用**。

### 核心原则

⚠️ **不要让 AI 帮你写完整教案**。如果你带回一份大而全的方案，大概率不会用。

✅ **要让 AI 帮你想清楚 1 件最小的事**——下学期第一节课你能做的尝试。

### 教学引入思考三问

```
Q1 · 我教的课程，学生最头疼 / 最被动的环节是什么？

Q2 · 如果给学生一个 AI 工具，这个环节会怎么变化？

Q3 · 把今天学的引入我的课，最小尝试是什么？
     （一节课就能开始的那种）
```

### 提示词模板

打开你的 AI 工具（DeepSeek / 豆包 / Kimi / OpenCode），复制下面这段**发给 AI**：

```
我是深圳职业技术大学的教师，我教【你的课程名】，
学生是【专业、年级、约多少人】。

今天我体验了 Vibe Coding (AI 自主写代码完成硬件项目) 和
零代码 AI 视觉两种 AI 工具。

我现在想认真想一下：如何把这套思路引入我的教学体系。

请先做一件事：**不要直接给我教学方案**。
而是先问我 3 个你最需要知道的问题，帮你理解我的
课程和学生的具体情况。

然后基于我的回答，我们一起讨论"我能做的最小尝试是什么"。

我会回答，然后我们继续往下聊。
```

AI 会先问你 3 个问题（比如"学生现在用什么工具做作业"、"你课上最难讲清的一个概念是什么"）。**来回聊 3-5 轮**，把"我的最小尝试"想清楚。

### 一时说不清楚你的课程？用这个模板填空

```
我教《____________》课程，
大 __ 年级，__________ 专业，
每周 __ 课时，学生约 __ 人。

学生平时作业主要是 _______________。

我课上最难讲清楚的一个概念或技能是 _______________。
```

### 最后 · 写下你的最小尝试 · 公开承诺

拿一张便签，写一句话：

```
我下学期的第一节课，
要在【课程名】里试做一次：________________
```

**拍照发到培训群** = 公开承诺。

讲师两周后 @ 每个人："承诺进度如何？" 我会继续帮你。

> **你不是带走一份完整教案**。
> **你带走的是一个起点**——下学期第一节课能做的第一步。

## 十三、卡住了怎么办

### 三步求助法

```
1. 先问 AI（3 分钟）
   把问题 / 报错截图给你的 AI，90% 能自己解决

2. 举手 / 直接喊助教
   助教立刻过来，5 个助教全程巡场

3. 和同桌互助
   很多问题你的同事能秒解
```

### 常见问题速查

| 问题 | 解决方法 |
|---|---|
| OpenCode 命令找不到 | 关闭终端重新打开；或找助教 U 盘安装 |
| 编译失败 | 告诉 AI："编译报错了，帮我看看" |
| 上传失败 | 按住 XIAO 的 **BOOT 按钮**再插 USB 线，进入下载模式 |
| Buzzer 不响 / 风扇不转 | 告诉 AI："[硬件名]没反应，帮我检查引脚和驱动方式" |
| 手机连不上 XIAO WiFi | 确认已上传 Web 控制程序；多人同名时 WiFi 名加自己姓氏 |
| 浏览器打不开 192.168.4.1 | 确认手机连的是 XIAO 的 WiFi（不是校园网） |
| SenseCraft 打不开 | 校园网可能受限，用手机热点 |
| AI 生成的代码有 bug | 把报错信息直接丢给 AI，它会自己修 |
| **电脑完全没法装软件** | **和同桌共用一台电脑，全程体验不打折** |
