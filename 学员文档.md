## 二、你的硬件套件

每人一套，培训结束后归还。

| 组件 | 说明 |
|---|---|
| XIAO ESP32S3 Sense | 主控开发板，拇指大小，内置摄像头+麦克风，支持 WiFi |
| Grove Base for XIAO | 扩展板，方便连接 Grove 模块 |
| Grove LED 模块 | 可控 LED 灯，接在扩展板 **D0** 端口 |
| Grove 风扇模块 | 可控小风扇，接在扩展板 **D1** 端口 |
| USB-C 数据线 | 连接电脑，供电+上传程序 |

### 硬件连接方式

```
电脑 ──USB-C──→ XIAO ESP32S3 ──插在──→ Grove Base 扩展板
                                          ├── D0 端口 → Grove LED
                                          └── D1 端口 → Grove 风扇
```

## 三、什么是 Vibe Coding？

**未来的关键能力不是"写代码"，而是"描述需求"。**

| 传统编程 | 聊天式 AI | **Vibe Coding** |
|---|---|---|
| 学语法、记函数 | 打开网页 AI，描述需求 | 写需求文档 |
| 自己写代码 | 复制 AI 的代码 | **AI 自动写代码** |
| 自己编译调试 | 粘贴到 IDE，手动上传 | **AI 自动编译上传** |
| 报错自己查 | 复制报错发给 AI 再粘贴回来 | **AI 自己看报错自己修** |
| 你是程序员 | 你是传话筒 | **你是导演** |

> Vibe Coding 这个概念由 Andrej Karpathy（前 Tesla AI 总监、OpenAI 联合创始人）在 2025 年提出："你完全可以沉浸在'感觉'里，看效果、说修改、再看效果，忘掉代码本身的存在。"

## 五、Vibe Coding 核心体验

### 第一幕：从空白到 LED 闪烁

讲师投影演示后，你跟着做：

```
> 请阅读 docs/requirements-blink.md，
  根据需求文档创建项目并生成代码，编译并上传到开发板。
```

AI 会自动：
1. 读需求文档（知道你要让 LED 闪烁）
2. 读 CLAUDE.md（知道硬件是 XIAO ESP32S3，LED 接在 D0）
3. 创建项目 → 写代码 → 编译 → 上传

**LED 闪了 = 成功。** 全程你没写一行代码。

---

### 第二幕：手机控制 LED 和风扇（全场高潮）

```
> 请阅读 docs/requirements-web-control.md，
  根据需求文档创建项目并生成代码，编译并上传到开发板。
```

AI 会自动生成一个完整的 Web 控制应用：
- XIAO 变成 WiFi 热点
- 上面跑一个网页服务器
- 网页有 LED 和风扇的开关按钮

**用手机体验：**

1. 手机打开 WiFi 设置
2. 搜索并连接 **XIAO-Lab**（密码 **12345678**）
3. 打开浏览器，输入 **192.168.4.1**
4. 点 LED 按钮 → LED 亮了！点 Fan 按钮 → 风扇转了！

> 这整个应用——从 WiFi 热点到网页界面到硬件控制——全部从一份需求文档自动生成。

## 六、扩展体验：零代码 AI 视觉

换一种 AI 赋能方式——不写代码也不写需求文档，纯图形界面操作。

### Step 1：部署预置模型

1. USB 重新连接 XIAO
2. 打开 https://sensecraft.seeed.cc
3. 选择"手势识别"预置模型
4. 一键部署到 XIAO
5. 对着摄像头举手 → **LED 亮了！**

### Step 2：训练你自己的 AI 模型

1. **采集图像**：对着摄像头拍"举手"图片 20 张 + "放下"图片 20 张
   （也可以自定义：竖大拇指/比心/OK 手势——什么都行）
2. **云端训练**：点"开始训练"→ 等待约 1 分钟
3. **部署+测试**：将训练好的模型部署到 XIAO → 实时测试识别效果

### Step 3：模型驱动硬件

- 识别"举手" → LED 亮
- 识别"放下" → 风扇启动

> **Vibe Coding = 写需求，AI 写代码。零代码视觉 = 采数据，AI 训模型。** 两种 AI 赋能路径，你的课上可以选择适合的那种。

## 八、反思实战：用 AI 设计你的教学方案

**今天最重要的环节——你用刚学的 Vibe Coding 思维，和 AI 聊出一份教学方案。**

打开你的 AI 工具（DeepSeek / 豆包 / Kimi / OpenCode），复制下方提示词，把【】里的内容改成你的信息：

### 提示词模板

```
我是深圳职业技术大学的教师，所属学院是【你的学院】，教研室是【你的教研室】，
教授【你的课程名称】课程，学生是【专业/年级，约多少人】。

今天我体验了柴火创客学院的 AI 赋能硬件培训，学到了两种 AI 工具：
1. Vibe Coding（AI CLI 编程）：用自然语言描述需求，AI 自动写代码、编译、上传到硬件。
   核心体验：我用手机网页控制了桌上的 LED 和风扇，整个应用由 AI 从零生成。
2. 零代码 AI 视觉：用 SenseCraft AI 平台训练图像识别模型，部署到 XIAO 硬件，无需写代码。

柴火创客学院有以下课程模块可供参考：
- M0 零基础智能硬件入门：Grove 传感器 + Wio Terminal 交互屏 + XIAO 视觉 AI，AI 辅助编程

请帮我设计一份教学方案，包含：
1. 我的课程如何融入 AI 硬件工具（选择最适合我学科的 1-2 个模块，并解释为什么适合）

请用 Markdown 格式输出，语言简洁专业。
```

### 不知道填什么？参考这些示例：

## 十、常见问题

| 问题 | 解决方法 |
|---|---|
| OpenCode 命令找不到 | 关闭终端重新打开；或找助教用 U 盘安装 |
| `pio run` 编译失败 | 告诉 AI："编译报错了，帮我看看" |
| 上传失败 | 按住 XIAO 的 BOOT 按钮再插 USB 线，进入下载模式 |
| 手机连不上 XIAO WiFi | 确认 XIAO 已上传 Web 控制程序；多人同名 WiFi 冲突时改 SSID |
| 浏览器打不开 192.168.4.1 | 确认手机已连上 XIAO 的 WiFi（不是校园 WiFi） |
| SenseCraft AI 打不开 | 校园网可能受限，用手机热点 |
| AI 生成的代码有 bug | 把报错信息告诉 AI，它会自己修 |
