## 三、什么是 Vibe Coding？

**Vibe Coding（氛围编程）** 由 Andrej Karpathy（前 Tesla AI 总监、OpenAI 联合创始人）在 2025 年提出：

> "你完全可以沉浸在'感觉'里，看效果、说修改、再看效果，忘掉代码本身的存在。"

| 传统编程 | 聊天式 AI | **Vibe Coding** |
|---|---|---|
| 学语法、记函数 | 打开网页 AI，描述需求 | 写需求文档 |
| 自己写代码 | 复制 AI 的代码 | **AI 自动写代码** |
| 自己编译调试 | 粘贴到 IDE，手动上传 | **AI 自动编译上传** |
| 报错自己查 | 复制报错发给 AI 再粘贴回来 | **AI 自己看报错自己修** |
| 你是程序员 | 你是传话筒 | **你是导演** |

每人一套，培训结束后归还。

| 组件 | 说明 |
|---|---|
| XIAO ESP32S3 Sense | 主控开发板，拇指大小，内置摄像头+麦克风，支持 WiFi |
| Grove Base for XIAO | 扩展板，方便连接 Grove 模块 |

### 三步接线

```
Step 1: XIAO 插入 Grove Base 扩展板（对准引脚插紧）
Step 2: Grove LED 模块插入 D0 端口
Step 3: Grove 风扇模块插入 D1 端口
最后: USB-C 数据线连接电脑
```

> XIAO 上有小灯亮起 = 供电正常。接不好？举手找助教。

## 六、渐进体验：四轮 Vibe Coding

**不需要写需求文档，不需要打开任何编辑器——直接跟 AI 说话。**

### Round 1：让 LED 亮起来

```
> 帮我为 XIAO ESP32S3 写一个程序，
  让连接在 D0 端口的 Grove LED 每隔 500 毫秒闪烁一次。
  编译并上传。
```

**LED 闪了 = 你的第一个硬件程序跑起来了。** 一句话搞定。

### Round 2：加上蜂鸣器

```
> LED 亮的时候，让板载蜂鸣器也同步鸣叫一声；
  LED 灭的时候，蜂鸣器停止。编译上传。
```

从"只有灯"变成"声光联动"。注意：AI 在已有代码的基础上**增量修改**——你不需要重新开始。

### Round 3：再加风扇

```
> 在现有基础上，让 D1 端口的 Grove 风扇也跟着 LED 同步转动：
  LED 亮时风扇转，LED 灭时风扇停。编译上传。
```

三句话，三轮迭代——从单个 LED 到声光风三者联动。

### Round 4：SOS 信号（来点有意思的）

```
> 把刚才的闪烁模式改成莫尔斯电码，
  持续发送声光 SOS 信号（短短短-长长长-短短短），
  LED + 蜂鸣器 + 风扇三者同步。编译上传。
```

你没学过莫尔斯电码？没关系——AI 知道。

> **四句话 · 四轮迭代 · 从 LED 闪烁到国际通用求救信号。** 全程没写一行代码。

## 七、需求文档驱动：Web 控制应用

刚才的四轮是"聊天模式"——一句话能说清的需求。但如果要做更复杂的东西呢？比如一个手机能控制的智能设备——一句话说不清了。这时候需要**需求文档**。

### AI 读需求文档，自动做一切

课程仓库里已经准备好了一份需求文档。让 AI 读它就行：

```
> 请阅读 docs/requirements-web-control.md，
  根据需求文档创建项目并生成代码，编译并上传到开发板。
```

AI 会自动：
- 读需求文档（知道你要什么）
- 读 CLAUDE.md（知道硬件是什么）
- 配置 WiFi AP 热点 → 生成 Web Server → 创建网页界面 → 编译上传

### 用手机体验

1. 手机打开 WiFi 设置
2. 搜索并连接 **XIAO-Lab**（密码 **12345678**）
3. 打开浏览器，输入 **192.168.4.1**
4. 点 LED 按钮 → **LED 亮了！** 点 Fan 按钮 → **风扇转了！**

> 这整个应用——WiFi 热点、Web 服务器、网页界面、硬件控制——全部从一份需求文档自动生成。你做了什么？你写了需求（其实连需求都是我们准备好的）。AI 做了剩下所有的事。

## 九、扩展体验：零代码 AI 视觉

换一种 AI 赋能方式——不写代码也不写需求文档，纯图形界面操作。

### Step 1：部署预置模型（5min）

1. USB 重新连接 XIAO
2. 打开 https://sensecraft.seeed.cc
3. 选择"手势识别"预置模型
4. 一键部署到 XIAO
5. 对着摄像头举手 → **LED 亮了！**

### Step 2：训练你自己的 AI 模型（10min）

1. **采集图像**：对着摄像头拍"举手"图片 20 张 + "放下"图片 20 张
   （也可以自定义：竖大拇指 / 比心 / OK 手势——什么都行）
2. **云端训练**：点"开始训练"→ 等待约 1 分钟
3. **部署+测试**：将训练好的模型部署到 XIAO → 实时测试识别效果

### Step 3：模型驱动硬件

- 识别"举手" → LED 亮
- 识别"放下" → 风扇启动

> **Vibe Coding = 写需求，AI 写代码。零代码视觉 = 采数据，AI 训模型。** 两种 AI 赋能路径，你的课上可以选择适合的那种。

## 十三、常见问题

| 问题 | 解决方法 |
|---|---|
| OpenCode 命令找不到 | 关闭终端重新打开；或找助教用 U 盘安装 |
| `pio run` 编译失败 | 告诉 AI："编译报错了，帮我看看" |
| 上传失败 | 按住 XIAO 的 BOOT 按钮再插 USB 线，进入下载模式 |
| 蜂鸣器不响 | 告诉 AI："蜂鸣器没声音，帮我检查引脚和驱动方式" |
| 手机连不上 XIAO WiFi | 确认已上传 Web 控制程序；多人同名 WiFi 时改 SSID 为你自己的名字 |
| 浏览器打不开 192.168.4.1 | 确认手机已连上 XIAO 的 WiFi（不是校园 WiFi） |
| SenseCraft AI 打不开 | 校园网可能受限，用手机热点 |
| AI 生成的代码有 bug | 把报错信息告诉 AI，它会自己修 |
